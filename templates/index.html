System Architecture

How it works

01
Cloud Environment
Simulates a cloud cluster with configurable servers. Poisson-distributed request arrivals, per-server queues, CPU utilization, and SLA violation tracking.
02
DQN Agent
Double DQN with experience replay and target network. Observes queue lengths, CPU usage, and server count. Actions: route to server N or scale up/down.
03
Baseline Comparison
Evaluated against Random, Round Robin, and Least Connections routing. Metrics: cumulative reward, avg latency, SLA violations, CPU utilization.
Baselines

Routing strategies compared

Random Routing
Assigns each request to a randomly selected server. Simple but ignores all load information — used as worst-case baseline.
Round Robin
Cycles through servers sequentially. Distributes evenly without considering actual load — good for uniform workloads.
Least Connections
Routes to the server with the smallest current queue. Greedy, reactive load balancing — strong classical baseline.
DQN Agent
Learns a policy via deep Q-learning that considers global state, future consequences, and the cost of SLA violations and server scaling.
