Evaluation Parameters
Test Episodes
▶ Description & Rules
What: How many episodes each method is evaluated over. More episodes give more stable statistics.
▸ Recommended: 100–500. Each method runs this many full episodes.
Episode Length (steps)
▶ Description & Rules
What: Timesteps per episode. Each episode resets the environment with freshly randomised server speeds.
Note: Should match the value used during training for a fair comparison.
▸ Range: 10+. Recommended: 200–1000.
Avg Requests / Step (λ)
▶ Description & Rules
What: Poisson arrival rate λ. Each timestep, Poisson(λ) requests arrive and must be routed.
Total CPU demand per step = λ × avg_workload.
▸ Range: 1+. Should match training for a fair eval.
Traffic Variance (λ ±)
▶ Description & Rules
What: Each episode, λ is re-sampled from [λ − var, λ + var]. Should match the value used during training.
Rule: Must be ≤ λ to prevent zero-request episodes. Set to 0 for fixed traffic.
▸ Set to 0 if the model was trained without traffic variance.
Avg Request Workload
▶ Description & Rules
What: Average CPU units each request requires from the target server.
Rule: Workload must be ≥ variance.
▸ Range: 1+. Recommended: 5–20.
Workload Variance (±)
▶ Description & Rules
What: Workload drawn from Uniform(avg−var, avg+var) — controls request size variability.
Rule: Must be ≤ avg_workload (otherwise negative workloads possible).
▸ Range: 0 – avg_workload. 0 = deterministic.
Avg Processing Power
▶ Description & Rules
What: Mean CPU units each server processes per step. Each server gets a unique speed drawn from Uniform(avg×0.3, avg×1.7) — this heterogeneity is exactly what DQN exploits over classical baselines.
▸ Recommended: set so N×power ≥ λ×workload (fleet can handle load).
Max Queue Length
▶ Description & Rules
What: Hard cap on each server's queue depth. Requests that would exceed this are clipped.
Rule: Should be at least 2× SLA threshold — otherwise SLA fires immediately under any load.
▸ Range: 10+. Recommended: 100–500.
SLA Latency Threshold
▶ Description & Rules
What: If average fleet queue depth exceeds this value at any step, an SLA violation is counted for that step.
Rule: Must be < max_queue_length (otherwise SLA can never trigger).
▸ Range: 1 – max_queue. Recommended: 40–100.
