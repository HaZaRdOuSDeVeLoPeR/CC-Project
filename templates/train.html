Set up the environment and DQN hyperparameters, then train the agent.

Environment
Server Count ✓
●
▶ Description & Rules
What: Number of servers in the fleet. Each gets a unique random processing speed.
Effect on action space: Action space = N servers. N=10 → 10 actions.
▸ Range: 2 – 50. Recommended: 5–15 for fast training.
Avg Requests / Step (λ)
●
▶ Description & Rules
What: Poisson arrival rate λ. Each step, Poisson(λ) requests arrive.
Total CPU demand per step = λ × avg_workload.
▸ Range: 1+. For full load: set λ so arrival ≈ fleet throughput.
Traffic Variance (λ ±)
●
▶ Description & Rules
What: Each episode, λ is re-sampled uniformly from [λ − var, λ + var]. Set to 0 for fixed traffic.
Why: Forces the agent to generalise across light and heavy traffic regimes rather than overfitting to one load level. DQN with variance > 0 will outperform baselines more robustly in real deployments.
▸ Range: 0 – λ. 0 = fixed λ each episode. Recommended: 30–50% of λ.
Avg Request Workload
●
▶ Description & Rules
What: Average CPU units each request requires.
Rule: Workload must be ≥ variance (variance cannot exceed workload).
▸ Range: 1+. Recommended: 5–20.
Workload Variance (±)
●
▶ Description & Rules
What: Workload is drawn from Uniform(avg−var, avg+var).
Rule: Must be ≤ avg_workload (otherwise negative workloads possible).
▸ Range: 0 – avg_workload. 0 = deterministic workload.
Avg Processing Power
●
▶ Description & Rules
What: Mean CPU units processed per server per step. Each server gets a unique speed drawn from Uniform(avg×0.3, avg×1.7) — this heterogeneity is why DQN outperforms classical algorithms.
Fleet throughput: N × avg_power CPU/step on average.
▸ Range: 1+. Recommended: set so N×power ≥ λ×workload.
Max Queue Length
●
▶ Description & Rules
What: Hard cap on each server's queue depth. Requests above this are dropped (clipped).
Rule: Should be at least 2× SLA threshold — otherwise SLA fires immediately when any server gets loaded.
▸ Range: 10+. Recommended: 100–500.
SLA Latency Threshold
●
▶ Description & Rules
What: If avg fleet queue depth exceeds this, an SLA violation penalty (−20) is applied that episode step.
Rule: Must be < max_queue_length (otherwise SLA can never trigger).
▸ Range: 1 – max_queue. Recommended: 40–100.
Episode Length (steps)
●
▶ Description & Rules
What: Number of timesteps per episode before environment resets and server speeds are re-randomised.
Rule: Longer episodes give the agent more time to learn within an environment instance but slow each training cycle.
▸ Range: 10+. Recommended: 200–1000.
DQN Hyperparameters
Training Episodes
●
▶ Description & Rules
What: Total episodes to train. Each episode resets the env with fresh server speeds.
Rule: More episodes = better convergence. Stop early if rolling avg reward plateaus for 50+ episodes.
▸ Recommended: 500–1000. Minimum: 200.
Discount Factor (γ)
●
▶ Description & Rules
What: How much the agent values future rewards vs immediate reward. γ=0.99 means reward 100 steps away is worth 0.99^100 ≈ 0.37 of today's reward.
Rule: Must be in [0, 1). Values ≥ 1 cause Q-values to diverge.
▸ Range: 0.9–0.999. Standard: 0.99.
Learning Rate
●
▶ Description & Rules
What: Adam optimiser step size. Too high = Q-values oscillate/diverge. Too low = very slow convergence.
Rule: 0.00001 – 0.01. For CPU use 0.0005; for GPU you can try 0.001.
▸ Standard: 0.0005 (CPU), 0.001 (GPU).
Batch Size
●
▶ Description & Rules
What: Number of transitions sampled from replay buffer per gradient step.
Rule: Must be strictly less than Replay Buffer Size. Larger batches = more stable gradients but slower per-step.
▸ Range: 32–512. CPU: 64–128. GPU: 256–512.
Replay Buffer Size
●
▶ Description & Rules
What: Capacity of the experience replay memory. Older transitions are discarded when full.
Rule: Must be > batch_size. Recommended to be at least 10× batch_size × episode_length / num_servers.
▸ Range: 5000–100000. Standard: 10000–25000.
Epsilon Start / Min
▶ Description & Rules
What: ε-greedy exploration rate. Agent takes random action with probability ε, greedy action with 1−ε.
Decays multiplicatively each episode: εt+1 = εt × (ε_min/ε_start)^(1/episodes).
Rules: Start must be > Min. Both must be in [0, 1]. Min should not be 0 (agent needs some exploration).
▸ Standard: Start=1.0, Min=0.05.
Target Network Update (steps)
●
▶ Description & Rules
What: Copy policy network weights → target network every N gradient steps. Target network provides stable Q-value targets.
Rule: Too small = unstable (target moves too fast). Too large = slow learning. Roughly 1–5× episode_length.
▸ Range: 100–2000. Standard: 500.
Hidden Layer Dim
●
▶ Description & Rules
What: Width of the two hidden layers in the DQN network. More neurons = more representational capacity.
Rule: Should be at least 2× state_size (= N×3+3). For N=10, state=33, so 66+ is minimum. 128 is standard.
▸ CPU: 64–128. GPU: 128–512.
